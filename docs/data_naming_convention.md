\# Edu Nexus – Data Naming Convention



This document defines strict naming rules for all input files stored in `data/raw/`.



\## Folder Usage



\- `data/raw/` → Original PDFs (syllabus, lab manuals, PYQs)

\- `data/processed/` → Cleaned text output generated by the pipeline

\- `data/artifacts/` → Chunks, FAISS index, BM25 index, graph exports, metadata



\## File Naming Format



Use this format for every PDF:



<dept>\_<subject>\_<type>\_<year>.pdf



\### Examples



\- cse\_dsa\_syllabus\_2025.pdf

\- cse\_dsa\_labmanual\_2025.pdf

\- cse\_dsa\_pyq\_2023.pdf



\## Allowed Values



\### dept

\- cse

\- it

\- aiml

\- etc



\### type

\- syllabus

\- labmanual

\- pyq



\## Rules



1\. No spaces in filenames

2\. Use lowercase only

3\. Use underscore (\_) as separator

4\. Keep short but meaningful subject names

5\. No duplicate files
